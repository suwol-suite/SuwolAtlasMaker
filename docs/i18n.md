# Localization

Suwol Atlas Maker localizes the Electron renderer and menu through shared
locale resources.

## Folder Structure

```text
src/shared/i18n/
  types.ts
  language.ts
  language-registry.ts
  locale-loader.ts
  format.ts
  menu.ts
  locales/
    en/
    ko/
src/renderer/i18n/
  index.ts
```

Each language folder must contain the same namespace files:

```text
common.json
menu.json
project.json
options.json
preview.json
sprites.json
metadata.json
batch.json
watch.json
errors.json
diagnostics.json
```

`en` is the fallback language. `ko` is a required supported language. The
language registry controls which locale folders are visible in the UI. A locale
folder can exist before it is enabled.

## Language Resolution

GUI settings store:

```ts
type AppLanguage = "system" | "en" | "ko";
```

System language resolves to `ko` for Korean locales such as `ko-KR`; other
locales resolve to `en`. Explicit `en` or `ko` always wins.

`src/shared/i18n/language-registry.ts` owns the enabled language list. The
renderer currently bundles English and Korean resources, so additional locale
folders should remain disabled until renderer resources and translations are
complete.

The renderer changes language immediately through `react-i18next`. It also
sends `app:setLanguage` to the Electron main process, which saves settings and
rebuilds the app menu.

## Key Policy

Keys are semantic, not sentence fragments:

```json
{
  "actions": {
    "export": "Export"
  }
}
```

Use namespaced keys in renderer code:

```tsx
t("common:actions.export")
```

Do not concatenate translated strings. Use interpolation instead:

```tsx
t("errors:validation.requiredField", { field: t("project:inputFolder.label") })
```

Allowed literals include CSS class names, internal enum values, file
extensions, JSON field names, and algorithm IDs such as `maxrects` or `shelf`.
New UI areas such as Quick Start, export result cards, recent items,
recommended settings, workspace reset, and batch set project lists must use
locale keys in both `en` and `ko`. Packing `.log.txt` output remains technical
and does not need translation.

## Korean Term Policy

Use these Korean terms consistently in UI strings:

- Atlas: `아틀라스`
- Sprite: `스프라이트`
- Preview: `미리보기`
- Export: `내보내기`
- Scan: `스캔`
- Project: `프로젝트`
- Profile: `프로필`
- Batch: `일괄`
- Status: `상태`
- Details: `자세히`
- Watch: `감시`
- Cache: `캐시`
- Trim: `트림`
- Crop: `크롭`
- Pivot: `피벗`
- Padding: `패딩`
- Extrude: `익스트루드`
- Quick Start: `빠른 시작`
- Recent Items: `최근 항목`
- Recommended Settings: `추천 설정`
- Workspace Reset: `화면 초기화`
- Result Card: `결과 카드`

Avoid user-visible wording such as `진단`, `인스펙터`, `메타데이터`,
`캐시 히트`, `워치 트리거`, `internal`, or `debug`. Technical namespace names,
file names, and schema terms may keep their established names where changing
them would break compatibility.

Buttons should stay short enough to avoid wrapping. Move longer explanations to
tooltips, panel guidance, or docs, and keep icon-only buttons supplied with
`title` and `aria-label`.

Menu labels are maintained in both `src/shared/i18n/menu.ts` and the `menu.json`
namespace so tests can verify parity. The View menu currently uses `Reset
Workspace`, `Reset Panel Sizes`, and `Reset Filters`; language preference is
not reset by these commands.

## Number And Date Formatting

Use helpers from `src/shared/i18n/format.ts` for locale-aware display:

- `formatDateTime`
- `formatNumber`
- `formatPercent`
- `formatFileSize`

Do not write formatted numbers into atlas JSON. Formatting is display-only.

## Adding A Language

1. Scaffold a hidden locale folder:

```bash
npm run i18n:add -- ja
```

2. Translate values only. Keep keys identical.
3. Add the language to `src/shared/i18n/language-registry.ts`.
4. Add renderer resources in `src/renderer/i18n/index.ts`.
5. Set `enabled: true` only after the locale is complete.
6. Run:

```bash
npm run i18n:check
npm run i18n:missing
npm run typecheck
npm test
```

`npm run i18n:check` validates namespace parity, nested key parity, JSON parse
errors, empty strings, and enabled language registry entries.

`npm run i18n:missing` lists missing namespace keys across locale folders. It
is useful while a hidden locale is still being translated.
